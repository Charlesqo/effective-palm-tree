# 环境信息与 UE（Unreal Engine）可安装性评估

## 1) 当前环境基础信息

通过命令检查得到：

- OS: Ubuntu 24.04.3 LTS
- Kernel: Linux 6.12.47
- CPU: 3 vCPU（Intel Xeon Platinum 8370C, KVM 虚拟化）
- Memory: 17 GiB（无 swap）
- Disk: 根分区 63 GiB，总可用约 33 GiB

## 2) 直接安装 UE 的可行性

### APT 可用性

`apt-cache search unreal` 仅返回了与 UE 无关的软件（例如 `slimevolley`），未发现官方 Unreal Engine 5/4 的 apt 包。

### 从源码构建的现实限制

在该环境中，从源码构建 UE 一般会遇到以下瓶颈：

1. **磁盘空间不足**：UE 源码 + 依赖 + 中间构建产物 + 最终产物通常需要远超当前剩余 33 GiB（常见需要 100+ GiB，构建期可能更高）。
2. **CPU 偏少**：3 vCPU 会导致编译极慢，常见需要更多核数才能在可接受时间内完成。
3. **图形/桌面环境限制**：当前环境是无完整桌面会话/无系统 DBus 的容器风格环境，图形工具链和编辑器运行体验受限。

## 3) 结论

- **可以尝试安装“启动器/辅助工具”**（之前已验证 Unity Hub 这一类工具可以安装到系统层）。
- **但在当前这台环境里，安装并可用地运行完整 UE 编辑器不可行性很高**，核心原因是 APT 无官方包 + 构建路径对磁盘/CPU/图形会话要求过高。

## 4) 若要真正安装 UE，建议的最低环境方向

- 至少 8~16 vCPU
- 至少 32~64 GiB 内存
- 至少 200 GiB 可用 SSD 空间（更稳妥 300 GiB+）
- 稳定的外网访问（含 Git LFS / 依赖下载）
- 完整图形会话（如果要启动编辑器 GUI）
